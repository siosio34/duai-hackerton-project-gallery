quality-gate-loop · A/B 비교

같은 초안, 두 가지 개선 방식

동일한 거친 초안 한 개를 ① 스킬 없이 LLM 단발 호출로, ② quality-gate-loop 스킬로 다듬었다. 결과물의 겉모습이 아니라, 무엇이 검증됐고 무엇이 드러났는지를 비교한다.

입력: AI 자동매매 봇 기획서 (41줄 초안) A: claude 1-pass B: 3-렌즈 게이트 루프 · 4 라운드

실험 설계

의도적으로 거칠게 쓴 v0 초안 하나를 동일하게 입력으로 주고, 두 경로로 처리했다.

공통 입력 (baseline)

AI 자동매매 봇 기능 추가 기획서 — v0

41줄 · "리스크는 유저 책임", "규제는 나중에 본다", "정확도 높이면 됨" 수준의 1차 초안
경로 A — 스킬 미사용

LLM 단발 개선

claude를 한 번 호출해 "잘 정리해줘". 평가도, 게이트도, 사용자 검증도 없음.

경로 B — 스킬 사용

quality-gate-loop

3개 렌즈(Claude 서브에이전트 / 외부모델 Codex+Grok / 한국인 페르소나 패널)로 채점 → 개선 → 재채점 루프.

결과물 — 나란히 보기

두 산출물 모두 초안보다 길고 깔끔해졌다. 차이는 "검증을 거쳤는가"에 있다.

경로 A · without_skill.md

겉보기엔 완성본

41줄 → 185줄. 목차·KPI 표·리스크 절·면책 부록까지 추가되어 보기 좋게 정리됨.

  • 문서 구조·가독성 즉시 향상
  • 리스크/규제 섹션이 형식적으로 채워짐
  • 아무도 "한국 사용자가 신뢰할까?"를 검증 안 함
  • 남은 빈틈(증거·신뢰)을 알 길이 없음
  • 점수·근거·정지 신호 전무
점수 없음 평가 자체를 돌리지 않았으므로 품질을 객관화할 수 없다.
경로 B · with_skill.md (=round 2)

점수 + 멈춤 이유

4라운드 동안 채점·개선을 반복하다, 텍스트로 못 고치는 신뢰 문제를 발견하고 정직하게 멈춤.

  • 3개 독립 렌즈가 교차 채점
  • 서술 완성도(native) 2.0 → 7.0 상승
  • 사용자 신뢰(persona)는 4.1 → 2.4 하락 — 게이트가 포착
  • 통과 강제 대신 에스컬레이션 STOP
  • 최고점 버전(round 2)을 최종본으로 복원
4.6 / 10 최종 판정 STOP. 부풀리지 않은 점수 + 사람이 결정할 항목 4건을 명시.

핵심 발견 — 렌즈 간 괴리

스킬을 쓴 경로에서만 보이는 그래프. 개선을 거듭할수록 한 렌즈는 오르고 다른 렌즈는 떨어졌다.

native (서술 완성도) external (Codex+Grok) persona (사용자 신뢰) 종합
왜 중요한가: 디테일·헤지 문구가 늘수록 서술 완성도는 올랐지만, 한국 일반 사용자 패널은 "리스크가 더 또렷이 보여서" 오히려 신뢰를 덜 했다. 단일 렌즈(예: LLM 자기채점)만 봤다면 "7점, 좋아짐"으로 끝났을 것을, 다중 렌즈가 괴리를 드러냈다. 그럴듯함 ≠ 신뢰.

라운드별 평가 (경로 B)

각 라운드마다 3개 렌즈가 점수와 "다음에 고칠 것"을 내놓는다. 페르소나는 게이트 점수가 아닌 advisory(사각지대 신호)로 본다.

왜 멈췄나 — 에스컬레이션

persona 렌즈가 3라운드 연속 floor(6.5) 미만에서 정체. 이건 윤문으로 못 고치는 설계·신뢰 문제라서, 루프가 사람에게 넘기고 멈춘다.

사람이 결정해야 할 항목

자동 개선기는 텍스트를 다듬을 뿐, 없는 증거를 만들거나 손실 책임 구조를 바꿀 수 없다. 아래는 루프가 식별해 넘긴 미해결 핵심:

    전체 문서 읽기

    세 버전의 실제 본문. 초안 → 두 갈래 결과물.

    // comparison/ai_trading_bot_기획서.md — 공통 입력
    // comparison/without_skill.md — claude 1-pass, 미검증
    // comparison/with_skill.md — 게이트 루프 최종본(최고점 round 2 복원)

    결론

    1-pass 개선은 결과물을 "완성된 것처럼" 보이게 만든다 — 그러나 빈틈을 알려주진 않는다. quality-gate-loop는 통과를 강제하지 않는다. 대신 실제 사용자 패널이 신뢰하지 않는 근본 이유를 숫자와 인용으로 드러낸 뒤, 텍스트로 못 고치는 지점에서 정직하게 멈추고 최고점 버전을 보존한다. 이 스킬의 값어치는 "점수를 올려주는 것"이 아니라 점수를 부풀리지 않고 결함을 드러내는 것이다.

    quality-gate-loop — 검증 오라클이 아니라 엄밀성 하네스. 실데이터 비교(comparison/): native=Claude 서브에이전트, external=Codex+Grok, persona=한국인 6인 패널(Nemotron-Personas-Korea).
    주의: persona 렌즈는 합성 프로필에 대한 LLM 역할극이라 실측이 아니며, 여기서는 정량 게이트가 아닌 advisory(사각지대 신호)로만 사용했다. 본 페이지는 단일 실행(run)의 실제 산출물을 시각화한 것이며 통계적 일반화가 아니다. 본 문서/예시는 투자·법률 자문이 아니다.
