# AI 자동매매 봇 기능 추가 기획서 (v1)

> 본 문서는 거래소에 "AI 자동매매 봇" 기능을 추가하기 위한 기획서다.
> 본 기획서의 수익률·성과 지표는 보장값이 아니며, 모든 수치 목표(KPI)·시장 데이터는 출처와 측정 방법을 명시한 항목과, 출시 전 확정해야 할 미확정(TBD) 항목으로 구분한다.
> **본 기능은 원금 손실이 발생할 수 있는 투자 보조 도구이며, 수익을 보장하지 않는다.**

## 0. 출시 불가 조건 (Go/No-Go 선행 게이트)
아래 중 하나라도 충족하지 못하면 출시하지 않는다.
- 규제·인허가 검토(2장) 미완료 또는 필요한 자격·라이선스 미확보
- 투자자 보호·고지·약관(3장) 법무 검토 미승인
- 리스크 관리 장치(5장: 손절·한도·킬스위치)의 실거래 검증 미완료
- AI 모델 검증(6장: 백테스트·페이퍼·파일럿)에서 사전 정의한 합격 기준 미달
- 자금·보안 설계(7장)의 보안 점검 및 사고 대응 절차 미승인

## 1. 배경 및 시장 근거
### 1.1 문제 정의
- 수동 매매 사용자는 24시간 시장 모니터링, 감정적 매매, 일관된 전략 실행의 어려움을 겪는다.
- 본 기능은 이 문제를 규칙 기반·검증된 전략의 자동 실행으로 보조하는 것을 목표로 한다.

### 1.2 타깃 사용자
- 1차: 거래 경험이 있으나 상시 모니터링이 어려운 기존 거래소 활성 사용자
- 2차: 자동화 전략에 관심 있는 신규 유입 사용자
- 초보자는 별도 보호 장치(기본 보수 전략, 한도 강제, 위험 고지 필수 확인)를 적용한다.

### 1.3 시장·경쟁·수요 근거 (출시 전 확정 필요)
근거 없는 "빨리 만들어야 한다"는 주장을 배제하고, 아래 항목을 데이터로 확정한 뒤 추진 여부를 결정한다.
- **경쟁사 기능 비교표**: 주요 경쟁 거래소의 자동매매/봇 기능 — 제공 전략, 수수료, 리스크 장치, 수익률 표시 방식 비교 (조사 출처·일자 명시)
- **목표 시장 규모**: 자사 활성 사용자 중 자동매매 관심 세그먼트 규모 — 내부 사용자 설문/행동 데이터 기반 산정
- **수요 검증**: 사전 인터뷰, 베타 대기자 신청 수, 유사 기능 클릭/문의 로그 등 측정 가능한 신호
- **차별점**: 위 비교를 근거로 "왜 우리 봇인지"를 1~2개 명확한 우위 항목으로 정의 (TBD)

## 2. 규제·법적 검토 (선행 항목)
규제 검토는 '나중'이 아니라 사업 성립의 전제다. 법무·컴플라이언스 검토를 출시 전 완료한다.
- **인허가 필요 여부 검토**: 자동매매 기능이 투자일임업·로보어드바이저(자동화 자문/일임) 등 인허가 대상에 해당하는지 판단. 해당 시 자격·등록 요건 확보 또는 비해당 구조 설계.
- **적용 법령 검토**: 자본시장법, 특정금융정보법(특금법), 가상자산이용자보호법 등 가상자산·금융 관련 법령의 적용 범위와 의무사항 확인.
- **광고·표시 규제**: 수익률·성과 광고 표시 규정, 과대광고·불완전판매 소지 차단.
- **준법 방안**: 위험 고지, 적합성 확인(사용자 투자성향·한도 설정), 약관 동의, 책임 범위를 제품에 반영. 규제 해석이 불확실한 항목은 법률 자문 결과를 근거로 의사결정.
- 본 검토 결과는 0장 출시 게이트의 통과 조건이다.

## 3. 투자자 보호 · 고지 · 책임 구조 (선행 항목)
"리스크는 유저 책임" 한 줄로 대체할 수 없으며, 다음을 명시한다.
- **원금손실 고지**: 자동매매는 손실이 발생할 수 있으며 수익을 보장하지 않음을 가입·실행 단계에서 명확히 고지하고 사용자 확인을 받는다.
- **적합성 확인**: 사용자의 투자 경험·위험 감내 수준을 확인하고, 그에 맞는 전략·한도만 허용. 초보자 보호 기본값 적용.
- **약관·동의**: 서비스 범위, 사용자 책임과 회사 책임의 경계, 면책 범위를 약관에 구체화하고 법무 검토를 받는다.
- **면책 범위**: 시장 변동에 따른 손실과, 회사 귀책(시스템 장애·오작동) 사고를 구분하여 책임·보상 기준을 분리 정의한다.

## 5. 리스크 관리 정책
손실·장애·오작동·시장 급변에 대한 통제 장치를 설계한다.
- **손절(Stop-loss)**: 전략별·포지션별 손절 기준 강제. 사용자 설정 가능하되 최대 허용 손실폭 상한 적용.
- **투자 한도**: 봇당/사용자당 최대 투입 금액 한도. 적합성 결과에 따라 차등.
- **킬스위치**: 사용자 즉시 중단 기능 + 회사 측 전체/개별 봇 강제 중단 기능.
- **최대 손실(Max Drawdown) 한도**: 정의된 MDD 초과 시 봇 자동 중단.
- **시장 급변 대응**: 급락·연쇄청산·이상 변동성 감지 시 신규 진입 차단 및 보호적 중단 절차.
- **장애·해킹·모델 오류 대응**: 사고 유형별 대응 플레이북, 사용자 통지, 정산·보상 기준(3장 면책 구분과 연계).

## 6. AI 모델 / 전략 설계 및 검증
"AI가 알아서 판단", "정확도 높이면 됨"은 실행 계획이 아니므로 다음으로 구체화한다.
- **전략 정의**: 단타·장기·DCA 등 제공 전략을 규칙으로 명시하고, AI/모델이 담당하는 의사결정 범위를 한정한다.
- **데이터**: 시세·체결·호가 등 데이터 출처와 학습 기간을 명시. 생존편향·룩어헤드 편향 배제 절차 포함.
- **피처**: 사용 입력 변수(가격/거래량/기술지표 등)를 정의.
- **검증 방법**:
  - 백테스트 — 수수료·슬리피지 반영, 과적합 방지(워크포워드/아웃오브샘플 분리)
  - 페이퍼 트레이딩(모의 실시간) — 백테스트와 실거래 간 괴리 확인
  - 실거래 파일럿 — 소액·제한 사용자 대상 단계적 검증
- **성능지표 및 한계**: 샤프 비율, 최대낙폭(MDD), 승률 등으로 평가하되 합격 기준을 사전 정의. 실패 사례·모델 한계와 "수익 보장 아님"을 명시.
- **운영 모니터링**: 모델 드리프트·성과 저하 감지 지표와 재검증/롤백 기준 정의.

## 7. 자금 · 보안 설계
- **API 권한 범위**: 봇은 거래 권한만 보유하고 출금 권한을 갖지 않도록 권한 최소화. 키 보관·폐기 정책 정의.
- **예치금/출금 통제**: 봇 투입 자금과 출금 경로 분리, 한도·인증 통제.
- **사고 대응·보상 정책**: 회사 귀책 사고(시스템 장애·오작동)와 시장 손실을 구분한 보상·정산 기준(3장 연계).
- **운영 절차**: 장애 및 시장 급변 시 운영 대응 절차와 책임자·에스컬레이션 경로 정의.

## 8. 수익률 표시 규정
- **산정식 명시**: 수익률 계산 방식과 기준 시점을 화면에 명확히 표기.
- **수수료 차감 기준**: 거래수수료·슬리피지를 반영한 순수익 기준으로 표시.
- **과거 수익률 오해 방지**: "과거 성과가 미래 수익을 보장하지 않음" 등 고지 문구를 함께 표시.
- 표시 내용은 2장 광고·표시 규제 검토를 통과해야 한다.

## 9. 사용자 흐름
온보딩부터 출금까지 완결되도록 구성한다.
1. 봇 메뉴 진입 → 위험 고지 확인 및 동의
2. 적합성 확인(투자성향·경험) → 허용 전략·한도 산정
3. 전략 선택 (전략별 설명·과거 검증 결과·리스크 안내 제공)
4. 자금 설정 (한도 내 투입, 손절·MDD 한도 설정)
5. 봇 가동 + 실시간 모니터링(수익률·포지션·리스크 지표)
6. 중단(킬스위치)/조정
7. 정산·출금

## 10. 기술·운영 범위 및 실행 계획
- **범위**: 전략 엔진, 모델 추론, 주문 실행, 모니터링/알림, 리스크 통제, 거래소 시스템(주문·체결·잔고 API) 연동.
- **인력**: 기획·개발(백엔드/모델)·데이터·QA·보안·법무/컴플라이언스 (역할별 배정 — TBD).
- **의존성**: 거래소 코어 시스템 연동, 데이터 파이프라인, 법무·규제 검토 결과.
- **수익 모델**: 수수료 기반. 표시·광고는 8장·2장 규정 준수.

## 11. 일정 (게이트 기반)
규제 검토·모델 검증·보안 점검 없는 "한두 달 MVP"는 채택하지 않으며, 단계별 게이트 통과를 전제로 한다.
- **0단계 — 검토·설계**: 시장·경쟁 분석(1.3), 규제·법무 검토(2장), 리스크/보안 설계(5·7장) 확정
- **1단계 — 모델·전략 검증**: 백테스트 → 페이퍼 트레이딩 → 합격 기준 충족(6장)
- **2단계 — 제한 파일럿**: 소액·제한 사용자 실거래 검증, 리스크 장치 실동작 확인
- **3단계 — MVP 출시**: 0장 출시 게이트 전 항목 통과 시
- **4단계 — 고도화**: KPI(4장) 기반 전략 확대·개선
- 각 단계 소요 기간은 0·1단계 결과에 따라 확정(TBD)하며, 임의 단축하지 않는다.
