01

요약

결론부터: 본 조사가 공개·검증 가능한 근거로 확인한 증거는 일관되게 한 방향을 가리킨다 — 실행은 자동화하되, 판단은 위임하지 말라. "LLM이 매매 판단으로 시장을 이긴다"는 주장은 동료심사 재평가에서 "평가 방법론의 산물"로 판정됐고[S22], 유일한 규모 있는 크립토 실거래 공개 실험에서는 프런티어 모델 6개 중 4개가 손실(합산 17일 −28%, 수수료가 손익 지배)을 냈다[S15]. 봇 수익이 공개 근거로 확인되는 경로는 펀딩비/베이시스 수확·마켓메이킹·가격차 차익 같은 알파 없는 실행 자동화이며, 거기에 LLM이 기여한다는 공개 증거는 없다 — 그 수익률조차 자본 유입으로 압축되는 중이다[S68].

그래서 개발자의 합리적 경로는: 성숙한 결정론 봇(Freqtrade·Hummingbot)의 페이퍼 트레이딩에서 시작하고, LLM은 리서치·코드 생성·시그널 "의견" 제공자로 한정하며[S7], 주문 권한과 키는 LLM 손이 닿지 않는 곳에 둔다 — 외부 텍스트 주입에 의한 무단 송금은 연구로 시연됐고[S32], 접근통제 침해로 악성 프롬프트가 주입돼 실제 자금이 탈취된 사고도 있었다[S33]. 한국 기준 자기 자금 봇 운용을 금지하는 명시 규정은 확인되지 않지만 반복 시장가 루프는 시세조종 고발 패턴이고[S47], 타인 자금 운용은 형사 리스크 영역이며, 과세는 세 차례 연기 끝에 2027-01 시행 예정이다[S52]. 가장 큰 캐비앗: 이 주제의 검색 결과는 거래소·벤더의 가짜 성과 수치로 오염돼 있다 — 본 보고서가 기각한 수치들을 §3.3에 명시했다[S63]. 투자 권유가 아니다.

02

핵심 인사이트

1실행은 자동화하되, 판단은 위임하지 말라 — 공개 근거가 일관되게 가리키는 방향

수익이 공개 근거로 확인되는 경로(펀딩비 차익, 마켓메이킹, 가격차 차익)는 전부 예측 없는 결정론적 실행 자동화이고, 이들 경로에 LLM이 기여한다는 공개 증거는 없다[S68][S69]. 확인된 기관 사례들의 LLM 활용도 리서치·문서 분석이지 주문 집행 위임이 아니다[S74].

2"LLM이 시장을 이긴다"는 주장은 편향 통제 후 소멸했다

동료심사 재평가(FINSABER, KDD 2026)는 상장폐지 종목 포함 20년 데이터에서 유명 LLM 에이전트들의 알파가 "평가 방법론의 산물"임을 보였고(FinMem Sharpe −0.228 vs 단순 보유 0.703)[S22], 유일한 규모 있는 크립토 실거래 공개 실험(Alpha Arena, 실자금)에서는 6개 모델 중 4개가 손실 — 합산 17일 만에 약 −28%였다[S15].

3반복 수렴하는 설계 패턴: LLM은 판단 보조, 주문과 리스크 한도는 결정론적 코드

학술 정식화[S2], Hummingbot의 Condor 설계(리스크 한도는 "에이전트가 절대 수정 못 하는 사용자 전용 가드레일")[S7], 개인 개발자 사례[S9]가 독립적으로 같은 구조에 도달했다. 2026 Q1에는 Kraken·OKX가 거래소 직영 MCP 인터페이스까지 출시했다[S8][S1].

4돈을 다루는 LLM 에이전트는 프롬프트 인젝션에 구조적으로 취약하다 — 실제 탈취 사례 존재

프린스턴 연구는 X 댓글 경유 간접 주입으로 ElizaOS 에이전트의 무단 ETH 전송을 시연했고 "프롬프트 기반 방어는 불충분"이라 결론지었다[S32]. AIXBT는 대시보드 무단 접근으로 악성 프롬프트가 큐잉되어 실제로 55.5 ETH(~$106K)를 탈취당했다 — 외부 텍스트 주입과는 경로가 다른 접근통제 사고지만, 프롬프트가 곧 공격면이라는 점은 같다[S33]. 2026년 OWASP 1위 등재와 에이전트 프레임워크 RCE CVE들로 이 위협 클래스는 오히려 커지고 있다[S73].

5한국 기준: 자기 자금 봇은 합법, 그러나 "초당 수회 반복 매수·매도" 패턴은 실제 고발 사례다

금융위는 2025-11 API 봇의 거래량 부풀리기 패턴을 시세조종으로 고발했다[S47]. 타인 자금 수탁 운용은 미신고 VASP(5년 이하 징역)·유사수신 영역에 근접하고[S49][S50], 가상자산 과세는 2027-01 시행이 확정 방침이다[S52].

6이 주제의 검색 결과 자체가 오염돼 있다 — 거래소 위키의 성과 수치를 믿지 말라

"2026 DeepSeek +126%" 같은 유통 수치는 검증된 어떤 실험과도 일치하지 않는 무일자 거래소 콘텐츠이며, 해당 거래소는 자체 AI 트레이딩 이벤트를 홍보 중이다[S63]. "알고 트레이더 60% 흑자" 류 벤더 수치도 전수 데이터(데이트레이더 97% 손실[S27])와 양립 불가다[S30].

03

상세 분석

여섯 개의 질문을 따라간다 — "AI 에이전트 봇"은 기술적으로 무엇이 다른가(§3.1), 개발자가 쓸 만한 도구는 무엇인가(§3.2), 실제로 돈이 된다는 근거가 있는가(§3.3), 무엇이 위험한가(§3.4), 한국에서 합법의 경계는 어디인가(§3.5), 그래서 무엇을 어떻게 시작해야 하는가(§3.6).

"AI 에이전트 봇"의 실체: LLM이 앉는 자리는 4가지이고, 주류는 판단 보조다

룰베이스/퀀트 봇은 고정 로직(지표·그리드·DCA)이나 클래식 ML 예측치를 사전 정의된 입력→주문 매핑으로 실행한다. "LLM 에이전트"가 다른 점은 인지 루프(perceive→memory→reason→act→learn)를 돌며 뉴스·소셜 같은 비정형 데이터를 읽고 자연어 근거와 함께 결정을 내린다는 것이다[S2]. 그러나 2026년 중반의 실전 구현을 보면 LLM이 앉는 자리는 네 단계로 갈린다: ① 아예 없음 — 가장 많이 쓰이는 FreqAI는 XGBoost류 클래식 ML이지 LLM이 아니다[S4]. ② 개발 도우미 — Jesse의 전략 코드 생성, 3Commas의 자연어 설정 어시스턴트처럼 런타임 밖에 있다[S11][S12]. ③ 시그널 생성기 — OctoBot의 ChatGPT 모드처럼 LLM의 "의견"을 받아 기존 결정론 엔진이 집행한다[S10]. 한국 개인 개발자들의 패턴도 같다(룰 필터 우선, 애매할 때만 GPT 호출)[S9]. ④ 디스크리션 보유 멀티에이전트 — 애널리스트·리서처·트레이더·리스크 매니저 역할극을 하는 TradingAgents가 대표지만, 정작 거래소 실행 계층이 없는 리서치 프레임워크다[S3].

주목할 것은 학술·실무·개인이 독립적으로 같은 아키텍처에 수렴했다는 점이다. 학술 서베이는 인지 계층과 실행·리스크 제약 계층의 분리를 정식화했고[S2], Hummingbot의 Condor(2026-04)는 확률적 LLM 계층과 결정론적 실행 계층을 MCP로 연결하되 포지션 한도·일손실·최대 드로다운은 "에이전트가 절대 수정할 수 없는, 사용자만 바꾸는 가드레일"로 박아 넣었다[S7]. 같은 risk-gated 패턴이 2026년 신규 설계들에서 반복된다[S14]. 즉 "LLM이 주문 API를 직접 만지는" 설계는 주류가 아니며, 그래야 할 이유도 §3.3–3.4에서 확인된다.

구조적 변화 하나: 2026년 3월 첫 주에 Kraken(Rust CLI, 134개 명령, MCP 내장, 페이퍼 트레이딩 엔진 동봉)[S8]과 OKX(agent-trade-kit MCP 서버)[S1]가 거래소 직영 에이전트 인터페이스를 출시했다. 데이터·실행 연결이 MCP로 표준화되면서 특정 에이전트 프레임워크에 대한 락인은 약해지는 중이다. 일부 2차 자료의 "Kraken CLI 2025-11 출시" 기술은 오류로, 공식 발표일은 2026-03-11이다[S8].

도구 지형: 성숙한 것은 결정론 봇이고, 인기 있는 것은 실행을 못 한다

GitHub 실측(2026-06-11) 기준 도구 지형의 역설이 뚜렷하다. 오픈소스 LLM 트레이딩 중 최다 스타인 TradingAgents(85.1k★)는 야후 파이낸스 티커 분석 프레임워크로 거래소 실행이 없고 자체 면책까지 명시한다[S3]. 반면 실거래가 가능한 성숙 봇들 — Freqtrade(51.3k★, 매일 커밋), Hummingbot(18.9k★), Jesse(8.0k★), OctoBot(6.1k★) — 은 모두 결정론적 코어에 ML/LLM을 보조 계층으로 붙이는 구조다[S1][S4]. 스타 수와 실전 채택은 다른 지표다.

계층 분담도 명확해졌다. ElizaOS(18.6k★)는 범용 에이전트 OS이고 트레이딩은 플러그인 유스케이스 중 하나다[S6]. Coinbase AgentKit은 "모든 AI 에이전트는 지갑이 필요하다"는 포지션의 지갑·온체인 실행 계층 전담으로, 시그널 기능이 없다[S5]. 상용(3Commas류)의 "AI"는 실체가 파라미터화된 룰 봇 + 자연어 어시스턴트 계층이다[S12]. 한편 토큰화 에이전트 런치패드인 Virtuals는 SDK가 3개월째 무푸시이고 간판 에이전트 AIXBT의 시총이 고점 대비 −97%로, 개발 도구로서의 성숙도는 낮다[S13].

거래소 정책은 우호적이다. 업비트는 공식 Open API와 rate limit을 문서화했고[S59], 빗썸은 자동매매를 아예 공식 서비스로 상품화했으며[S60], 바이낸스·코인베이스도 네이티브 봇과 API를 제공한다[S61]. 공통 기조는 "봇 허용 + rate limit 통제 + 제3자 봇 책임은 이용자 귀속"이다. 단 거래소의 봇 상품화는 수수료 사업이지 수익성 보증이 아니라는 점을 §3.3에서 다시 본다.

성과의 진실: LLM 알파를 입증하는 공개 근거는 없고, 수익이 확인되는 것은 비-LLM 실행 자동화다

낙관적 수치는 전부 백테스트다. TradingAgents는 3개 기술주 3개월 백테스트에서 누적 +23.21%, Sharpe 5.6–8.2를 보고했고[S20] FinMem은 TSLA 단일 종목 +61.8%를 보고했지만[S21], 실거래 이력은 0건이다. 그리고 이 주장들은 2025–2026년 편향 통제 재평가에서 붕괴했다. 동료심사를 거친 FINSABER(KDD 2026)는 상장폐지 종목을 포함한 2004–2024년 데이터로 재평가해 FinMem의 Sharpe가 −0.228(단순 보유 0.703)로 추락함을 보였고, "LLM 유래 알파는 좁고 편향된 평가의 방법론적 산물일 가능성이 높다"고 결론지었다[S22]. 이 논문은 ACM SIGKDD 2026 프로시딩 수록이 확인됐고 반박 연구는 발견되지 않았다 — 단 검증 범위는 미국 주식이므로, 크립토에 대한 직접 증거는 아래 실거래 실험과 묶어 읽어야 한다[S62]. 원인은 구조적이다: 사전학습 윈도 안의 백테스트는 예측이 아니라 회상이고[S23], 77편의 관련 연구 중 거래비용을 명시한 것은 1편, 완전 재현 가능한 것은 0편이다[S2]. 티커 익명화로 암기를 차단한 연구에서도 LLM 피처의 기여는 미미했다[S24].

실거래 증거는 더 직접적이다. 유일하게 규모 있는 크립토 실거래 공개 실험인 Alpha Arena 시즌 1(2025-10-18~11-04, 모델당 실자금 $10,000, Hyperliquid 무기한선물, 수수료 포함)에서 6개 프런티어 모델 중 4개가 손실을 냈다 — Qwen3 Max +22.3%, DeepSeek +4.9%였지만 GPT-5 −62.7%, Gemini −56.7%, 합산 약 −28%/17일이었고 주최측 스스로 "초기 구간 손익은 거래 비용이 지배했다"고 인정했다[S15][S17][S31]. 시즌 1.5(미국 주식)에서는 9개 중 1개만 흑자였고, 시즌 1의 "중국 모델 우위" 서사는 즉시 뒤집혔다[S18] — 경로도 불안정해서 DeepSeek는 +48%에서 4일 만에 +4.9%로[S16], 시즌 1 선두 Qwen은 마지막 주에 피크 수익의 2/3를 반납했다[S65][S64]. 17일 × $10k 표본의 순위는 스킬이 아니라 노이즈에 가깝고, 라이브 벤치마크 연구도 "모델 백본보다 스캐폴드가 결과를 좌우한다"고 본다[S25]. 다른 라이브 실험도 같은 그림이다 — 예측시장 6일 실험에서 흑자는 7개 모델 중 2개였다[S26]. 이 실험들에서 노이즈와 신호를 구분하면: 모델 간 순위는 시즌마다 뒤집히는 노이즈지만, "수수료가 손익을 지배했고 다수 모델이 손실"이라는 합산 패턴은 계좌 전체에서 반복된 견고한 신호다 — 본 보고서가 증거로 쓰는 것은 후자뿐이며, 그것도 "단기·고비용 조건에서 LLM 판단이 비용을 극복하지 못했다"는 범위로 한정된다.

불확실 + 오염 경보 — Alpha Arena 시즌 2의 최종 결과는 본 조사에서 1차 확인하지 못했다(사이트 접근 차단)[S66]. 한편 검색 상위에 노출되는 "2026 DeepSeek +126%" 류 리더보드는 검증된 어떤 시즌과도 일치하지 않는 무일자 거래소 콘텐츠로, 자체 AI 트레이딩 이벤트를 홍보하는 이해관계가 있다 — 인용 불가 판정[S63].

그렇다면 봇으로 돈을 버는 사람은 없는가? 있다 — 단 본 조사가 공개 근거로 확인한 범위는 전부 알파 없는 실행 자동화다(비공개 퀀트 전략의 존재를 부정하는 것이 아니라, 검증 가능한 증거가 있는 범위가 그렇다는 뜻이다). 펀딩비/베이시스 수확은 온체인으로 관측되는 검증된 수익원이지만 2024년 평균 ~19%에서 2026년 2분기 한 자릿수로 압축됐고(자본 유입에 따른 자기소멸성)[S68], 김치 프리미엄 차익은 학술 전수 데이터로 평균 4.73%가 기록된 역사적 사례다[S69]. 마켓메이킹 수익은 존재하나 감사된 사용자 손익 공개는 없다[S67]. 이 니치들의 공통점: 예측이 없고, 수익원이 명시적이며(펀딩비·스프레드·가격차), LLM이 기여한다는 증거가 0건이다. 동시에 이것은 "쉬운 돈"이 아니다 — 펀딩비 수확은 음(−)펀딩 구간(2022년 평균 −6%)의 꼬리 리스크와 거래상대방 리스크를 지는 리스크 프리미엄이고[S68], 마켓메이킹은 전문 MM·펀드와의 경쟁 속에 "튜닝 후에만, easy money 아님"이라는 것이 커뮤니티의 실측 평가이며[S67], 개인 자본 규모에서는 인프라·모니터링 운영 부담 대비 절대 수익이 작을 수 있다. 기저율도 기억해야 한다 — 크립토 봇 사용자의 감사된 수익 통계는 어느 거래소에도 없으며(부재 자체가 발견)[S30][S75], 인접 전수 데이터에서 지속 데이트레이더의 97%가 손실이었고[S27][S28], 레버리지 리테일 상품(CFD)의 계좌 손실 비율 74–89%가 지금도 브로커 표준 경고문의 근거다(2018 자료, stale)[S29]. 단 이 통계들은 방향성 단타 매매 기준이다 — LLM 방향성 베팅의 기저율 참고치로는 적절하지만, 전략 구조가 다른 차익·마켓메이킹형 봇에 그대로 적용해선 안 된다(그쪽은 비교할 전수 데이터 자체가 없다).

상충 — 벤더 콘텐츠는 "알고 트레이더의 60%가 흑자"를 주장하지만[S30], 방법론 공개 전수 데이터(97% 손실[S27], 지속 흑자 <1%[S28])와 양립 불가능하다. 봇 구독·제휴 수익이라는 이해관계를 감안해 본 보고서는 전수 데이터를 채택한다.

리스크: 인젝션은 실증됐고, 수수료는 부호를 뒤집으며, 사기가 가장 큰 손실원이다

기술 리스크. 돈을 다루는 LLM 에이전트의 중심 위협은 신뢰할 수 없는 외부 텍스트(뉴스·소셜·툴 출력) 경유 프롬프트 인젝션이다. 프린스턴 연구(2025-03)는 X 댓글을 통한 간접 주입으로 ElizaOS 에이전트의 메인넷 무단 ETH 전송을 시연했고, 오염된 대화 이력이 플랫폼 간 전파되는 메모리 인젝션이 인증 게이트를 우회함을 보이며 "프롬프트 기반 방어는 불충분하다"고 결론지었다[S32]. 실전 사례도 있다: AI 에이전트 AIXBT는 대시보드 무단 접근으로 악성 프롬프트가 큐잉되어 55.5 ETH(~$106K)를 탈취당했다(2025-03)[S33] — 팀의 "AI가 속은 게 아니다" 해명은 보도된 공격 경위와 모순되어 채택하지 않는다[S72]. 2026년 들어 이 위협 클래스는 OWASP 1위 등재, 에이전트 프레임워크의 인젝션→RCE CVE 공개로 오히려 확대됐다[S73]. API 키 관리의 전범은 3Commas 유출(확인된 피해만 $14.8M)이며[S37], 표준 방어는 출금 권한 비활성 키 + IP 화이트리스트 + 도구별 키 분리·로테이션이다[S38].

시장 리스크. 백테스트 과최적화(아웃오브샘플에서 Sharpe 40% 이상 하락이면 의심)와 수수료·슬리피지가 핵심이다 — 수수료 반영만으로 백테스트 +180%가 실손실로 뒤집힌 사례가 교육 자료에 기록돼 있고(C급, 방향성)[S43], Alpha Arena 실거래에서도 수수료가 손익을 지배했다[S15]. 온체인 에이전트는 MEV 샌드위치 공격(이더리움 연 ~$60M 규모, 8초 만에 $215K 손실 사례)에 노출된다[S42].

사기 리스크 — 문서화된 대형 피해가 가장 집중된 영역. "AI 자동매매 + 원금보장"은 글로벌 공통 사기 공식이다. 해외는 Mirror Trading International($1.7B, 2.3만 명)[S34]과 2025년 SEC 집행 사례들($14M, $12.3M)[S35][S36], 국내는 가짜 수익 앱을 쓴 파파코 사건(피해 230억 원, 280명, 2025-07 기소)[S39]과 부산 리딩방 조직(245억 원, 118명 검거)[S40]이 대표적이고 금감원이 소비자 경보를 발령했다[S41]. 레드 플래그: 수익 보장, 100% 승률, 인플루언서+비공개 채팅방 퍼널, 소액 초기 출금 허용 후 증액 압박, 출금 마찰[S34].

문서화된 완화책. 페이퍼 트레이딩(Freqtrade dry-run), 페어당 포지션 한도, 손익 임계 킬스위치(Hummingbot), 텔레그램 수동 개입[S44]; LLM 에이전트 전용으로는 프롬프트 방어에 의존하지 말 것 — 주소 화이트리스트 + 송금의 대역외 인간 승인 + 키를 에이전트 손이 닿지 않는 인증 도구 뒤에 두기[S32], 포지션·상태는 LLM 메모리가 아닌 구조화 저장소에[S45], 리스크 한도는 사용자 전용 설정으로[S7].

규제(한국 중심): 봇 자체를 금지하는 규정은 없고, 경계는 "행위"와 "타인 자금"에서 그어진다

한국에는 자기 계정·자기 자금으로 봇을 돌리는 것을 금지하는 법령이 확인되지 않으며, 거래소들이 공식 API를 제공하고 빗썸은 자동매매를 상품화까지 했다[S59][S60]. 규제는 "봇이냐"가 아니라 "무슨 행위냐"로 작동한다. 가상자산이용자보호법(2024-07 시행)의 시세조종 금지(1년 이상 징역, 이익 3–5배 벌금)가 핵심인데[S46], 금융위는 2025-11-05 고발에서 "API를 통해 1초당 수회 시장가 매수·매도를 반복해 거래량을 인위적으로 부풀리는" 수법을 적시했다[S47]. 봇이 의도 없이 만들어내는 자전성 루프도 외형상 같은 패턴이 될 수 있으므로, 자기 주문 교차·호가 견인형 반복 매수는 설계 단계에서 배제해야 한다. 집행은 본격화됐다(첫 과징금 2025-09)[S48].

경계가 확실히 넘어가는 곳은 타인 자금이다. 고객 가상자산을 받아 운용·수익 배분하면 미신고 가상자산사업자(특금법, 5년 이하 징역)[S49][S50], "원금 보장+고수익" 모집은 유사수신에 해당할 소지가 크다. 자본시장법의 투자일임 규제는 '금융투자상품' 전제라 코인에 깔끔히 적용되지 않는데, 이는 합법이라는 뜻이 아니라 피해자 보호 장치가 없는 사각지대라는 뜻이다[S51]. 실무 맥락을 보태면, 본 조사에서 확인된 공개 집행 사례는 원금보장·자금 수탁형 사기에 집중되어 있고[S39][S40], 자산을 수탁하지 않는 시그널·API 연동형 서비스의 제재 사례는 확인되지 않았다 — 이는 안전하다는 보증이 아니라 당국 해석이 비어 있다는 뜻이다. 미국도 같은 방향이다 — SEC는 2025-02 신설 조직의 우선순위로 "AI washing"을 지정하고 AI 봇 표방 사기를 연쇄 집행했다[S53][S54]. EU는 MiCA가 서비스화를 인가 대상으로 묶지만 AI Act는 트레이딩을 고위험으로 분류하지 않는다[S55].

불확실 (미해결 법적 쟁점) — 자산을 수탁하지 않는 "시그널 판매 + 사용자 API 키 연동형" 모델의 법적 지위는 명확한 당국 해석을 확인하지 못했다[S51]. 또한 가상자산 과세(연 250만 원 초과 22%)는 세 차례 연기 끝에 2027-01-01 시행이 현 정부 방침이나[S52], 연기 전력을 감안하면 2026년 7월 세법개정안 발표 시 재확인이 필요하다. 본 보고서는 법률 자문이 아니다.

덧붙여 "AI 에이전트 토큰"과 "AI 에이전트 도구"를 구분해야 한다. ai16z 토큰은 시총 약 $2.6B 고점에서 약 $5.7M로 99% 이상 붕괴했고 "자율 AI가 실제로는 사람이 운영됐다"는 집단소송(원고 측 일방 주장)까지 제기됐으며[S57][S56], VIRTUAL도 고점 대비 −88.7%다[S58]. 그러나 ElizaOS 프레임워크 자체는 토큰 없이 쓰는 오픈소스다 — 토큰 시장의 붕괴는 투기 수요의 소멸이지 도구 평가의 대체물이 아니며, 다만 이 분야 "자율 AI" 마케팅 전반의 신뢰성에 대한 경고로 읽어야 한다.

개발자 행동 지침: 무엇으로 시작하고, 무엇을 절대 하지 말 것인가

시작 경로(증거 기반 순서). ① 성숙한 결정론 봇(Freqtrade 또는 Hummingbot)으로 페이퍼 트레이딩부터 — dry-run은 실주문 없이 전체 루프를 검증한다[S44]. ② 전략은 수익원이 명시적인 비-알파 니치(펀딩비/베이시스, 스프레드)에서 출발하되, 수익률이 자본 유입으로 압축되는 자기소멸성을 전제하라[S68]. ③ LLM은 리서치·전략 코드 생성·로그 해석 보조로 쓰고[S74][S11], 시그널에 쓰려면 OctoBot처럼 결정론 셸 안의 의견 제공자로 한정하라[S10]. ④ 실자금 전환은 소액 + 출금 비활성 API 키 + IP 화이트리스트 + 킬스위치를 갖춘 뒤에[S38][S44]. 백테스트는 수수료·슬리피지 반영과 아웃오브샘플 검증 없이는 숫자가 아니다[S43][S22].

금지 목록. LLM에 주문 전권과 키를 함께 주는 설계(인젝션 한 번이 곧 자금 손실[S32]), 자기 주문이 교차하는 워시트레이딩성 루프(시세조종 고발 패턴[S47]), 지인 포함 타인 자금 운용(미신고 VASP·유사수신 리스크[S49] — 국내 장기 운용자 후기의 일관된 교훈이기도 하다[S71]), 그리고 "AI 봇 수익 보장" 상품의 구매와 판매(전자는 사기 피해, 후자는 형사 리스크[S39][S36]).

현실적인 기대치. 국내 개발자들의 장기(1.5–3년) 자작 봇 운용 후기의 모달 결과는 손익분기 부근이며, 이는 생존편향을 감안하면 낙관적인 추정이다[S71]. 자동매매를 "수익 기계 만들기"가 아니라 "거래 규율의 자동화 + 엔지니어링 학습"으로 접근하는 것이 증거와 정합하는 기대 설정이다. 관찰 지표: Alpha Arena 시즌 2 결과(도구를 쥐여준 LLM도 손실인지)[S66][S19], 거래소 1st-party 에이전트 인터페이스의 확산[S8], 2026-07 세법개정안[S52].

두 표가 함께 말하는 것: 도구의 성숙도와 수익성은 별개의 축이다. 가장 성숙한 도구(Freqtrade)도 수익을 보증하지 않고, 가장 인기 있는 LLM 프레임워크(TradingAgents)는 실행조차 못 한다. 수익이 검증된 행은 모두 "예측 없음 + 결정론 실행 + 명시적 수익원"이고, LLM이 필요한 행이 하나도 없다 — 이것이 §3.3 결론("실행은 자동화, 판단은 비위임")의 표 형태다. 아키텍처의 세련됨(Condor류 risk-gated 설계)을 수익성 증거로 읽지 않도록 주의해야 한다[S7][S22].

05

리스크와 기회

리스크

  • 프롬프트 인젝션 → 자금 탈취 영향 치명·실증됨 — 연구 시연(ElizaOS 무단 전송)과 실전 사례(AIXBT 55.5 ETH) 모두 존재, 프롬프트 방어만으로 해소 불가[S32][S33].
  • 수수료·슬리피지의 손익 반전 발생 흔함 — 실거래 실험에서 수수료가 PnL을 지배했고, 백테스트 양수익이 실손실로 뒤집힌 사례[S15][S43].
  • 시세조종 패턴 형사 리스크 — 봇의 반복 시장가 매수·매도 루프가 금융위 고발 패턴과 외형상 일치 가능[S47].
  • 타인 자금 운용의 법적 경계 — 수탁 운용은 미신고 VASP(5년 이하)·유사수신 영역[S49][S50].
  • "AI 봇" 사기 생태계 — 국내만 수백억 원대 기소(파파코 230억, 부산 245억); 검색 결과의 성과 수치 오염 진행 중[S39][S40][S63].
  • API 키 위탁/유출 — 3Commas 전례 확인 피해 $14.8M[S37].

기회

  • 성숙한 오픈소스 스택 — Freqtrade·Hummingbot 등이 페이퍼 트레이딩·킬스위치·포지션 한도까지 내장해 안전한 실험 비용이 낮음[S1][S44].
  • 거래소 1st-party 에이전트 인터페이스 — Kraken CLI·OKX MCP 킷이 페이퍼 모드 동봉으로 출시, 연동 표준화(MCP)로 락인 약화[S8][S1].
  • 비-알파 니치의 검증된 수익원 — 펀딩비/베이시스·스프레드는 수익 구조가 투명해 엔지니어링으로 접근 가능(압축 중임을 전제)[S68].
  • LLM의 확실한 효용 영역 — 전략 코드 생성·리서치·로그 해석은 기관에서도 검증된 사용 패턴[S74][S11].
  • 규제 명확화의 진행 — 행위 기준 집행(시세조종)과 과세 일정 확정으로 "합법적 개인 운용"의 경계가 오히려 선명해짐[S47][S52].
06

결론과 전망

여섯 개 핵심 질문에 대한 직답:

  • 기존 봇과 뭐가 다른가? 비정형 데이터를 읽는 인지 루프가 차이지만, 실전 주류는 "LLM=판단 보조, 결정론 코드=집행+리스크 한도"의 분리 구조다. LLM에 주문 전권을 주는 설계는 주류도 아니고 안전하지도 않다[S7][S32].
  • 뭘 쓰면 되나? 실거래는 Freqtrade/Hummingbot, LLM 실험은 결정론 셸(OctoBot 패턴)이나 거래소 페이퍼 모드(Kraken CLI)에서. TradingAgents류는 학습용 — 실행이 없다[S3].
  • 돈이 되나? LLM 알파는 공개·재현 가능한 근거 기준으로 입증된 바 없다 — 주식 백테스트는 편향 통제 시 소멸했고[S22], 크립토 실거래는 단기·고비용 조건에서 다수 손실이었다[S15]. 공개 근거로 수익이 확인되는 것은 비-LLM 실행 자동화 니치뿐이고, 그마저 압축 중이다[S68].
  • 가장 큰 리스크는? 기술적으로는 인젝션+키 관리, 경제적으로는 수수료, 금액 기준으로는 사기다 — 셋 다 실측 사례가 있다[S33][S37][S39].
  • 합법인가? 자기 자금 운용을 금지하는 명시 규정은 확인되지 않았다. 경계선은 워시트레이딩성 패턴과 타인 자금이고, 거래소 약관·rate limit 준수는 별도 전제다. 2027년부터 과세를 전제로 기록 체계를 갖춰라[S47][S52].
  • 그래서 시작해도 되나? "엔지니어링 학습 + 거래 규율 자동화"가 목적이면 페이퍼 트레이딩부터 시작할 가치가 충분하다. "AI가 돈을 벌어준다"가 기대라면 증거가 반대를 가리킨다 — 장기 운용자들의 모달 결과는 손익분기 부근이다[S71].

결론이 바뀌는 조건: ① Alpha Arena 시즌 2(웹검색·도구를 쥔 LLM)가 수수료 차감 후 지속 흑자를 보이면 "LLM 알파 없음" 판정을 재검토해야 한다[S19][S66]. ② 거래소가 감사된 봇 사용자 수익 통계를 공개하면 기저율 추정이 달라진다[S75]. ③ "시그널 판매+사용자 키 연동" 모델에 대한 당국 해석이 나오면 서비스화 판단이 명확해진다[S51].

이 보고서는 투자 권유·법률 자문이 아니며, 인용된 모든 수치는 출처의 시점·조건과 함께 읽어야 한다.
